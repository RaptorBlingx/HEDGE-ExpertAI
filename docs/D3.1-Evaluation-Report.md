## 4. Methodology

- **Script:** `evaluation/evaluate.py`
- **Query set:** `evaluation/test_queries.json` — 69 natural-language queries
  (53 primary + 16 edge-case) spanning all 8 SAREF domains (Energy, Building,
  Environment, Agriculture, Water, City, Health, Manufacturing).
- **Metrics computed per query:**
  - **Precision@2:** Fraction of top-2 results that appear in the expected set.
  - **Recall@5:** Fraction of expected apps found in the top-5 results.
  - **Reciprocal Rank (RR):** 1/rank of the first relevant result.
  - **Latency:** Wall-clock time from request to response (seconds).
- **Aggregation:** Arithmetic mean for P@2, R@5, MRR; median and P95 for latency.
- **Gateway endpoint:** `POST /api/v1/apps/search` via `http://localhost:8080`.

## 6. Analysis

### 6.1 Strengths

- **Zero misses:** Every query returned at least one relevant result in its top-2
  (no P@2 = 0).
- **Near-perfect MRR (0.991):** The first relevant result is almost always at
  rank 1. Only Q53 ("Environmental sensing and monitoring") had RR = 0.50
  (relevant result at rank 2).
- **Exceptional latency:** Median 50 ms, P95 80 ms — 100× faster than the 5 s
  target. The hybrid search (vector + keyword) pipeline is highly efficient.
- **High recall:** 90.9 % of expected apps appear in the top-5 results.

### 6.2 Improvement Opportunities

- **Cross-domain queries (Q52, Q53):** Broad queries like "IoT apps for smart
  home" and "Environmental sensing and monitoring" have lower recall (0.33)
  because expected apps span multiple SAREF classes. Multi-class boosting or
  query expansion could help.
- **Secondary-match precision:** 31 queries score P@2 = 0.50, meaning the
  second result is not in the expected set. This is partly a test-set artefact
  (expected sets have only 1–3 ground-truth apps) and partly a ranking gap for
  closely-related domains.
- **Re-ranker potential:** With `RERANKER_ENABLED=false`, the current pipeline
  relies solely on cosine similarity. Enabling a cross-encoder re-ranker would
  likely push P@2 above 80 %.

## 9. Conclusion

The HEDGE-ExpertAI system **passes** all defined KPIs. Search precision at 71.7 %
meets the ≥ 70 % target, and response latency at 50 ms median is well within the
< 5 s requirement. The system reliably surfaces relevant IoT applications across
all SAREF ontology domains with a near-perfect MRR of 0.991.

**Recommendation:** The system is ready for integration testing with the live
HEDGE-IoT App Store API. The re-ranker feature (currently disabled) offers a
clear path to further precision gains if needed.
